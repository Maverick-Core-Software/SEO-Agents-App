# Agent Instructions

## Role
You are working inside a local AI workflow project for Grizzly Electrical Solutions.

## Operating Rules
- Inspect files before editing.
- Make one focused change at a time.
- Do not rewrite unrelated files.
- Do not delete files unless explicitly asked.
- Explain what changed after edits.
- Prefer simple, maintainable code over clever abstractions.
- Keep outputs practical and business-useful.

## Safety Rules
- Do not modify .env unless explicitly asked.
- Do not expose API keys, tokens, credentials, or private customer data.
- Do not run destructive commands.
- Ask before installing packages.
- Ask before changing architecture.

## Project Goals
- Build useful SEO/local presence automation agents.
- Prioritize workflows that save time, create content, improve GBP/local rankings, or reduce manual admin work.
- Keep the system understandable enough to maintain later.

## Preferred Workflow
1. Read relevant files.
2. Summarize what exists.
3. Propose a small next step.
4. Wait for approval before larger edits
